Web 安全 ​

Web 安全涉及保护网络应用免受恶意攻击的技术和实践。包括防护常见漏洞、实施安全策略、加密敏感数据。

常见攻击类型 ​

XSS (跨站脚本攻击) ​

  • 原理: 注入恶意脚本、在受害者浏览器执行
  • 类型: 反射 XSS、存储 XSS、DOM XSS
  • 危害: 会话劫持、数据盗取、页面篡改

防护方法 ​

  • 输入验证: 白名单验证、拒绝非预期输入
  • 输出编码: HTML/JavaScript/URL 编码、转义特殊字符
  • CSP: 内容安全策略、脚本来源限制
  • 框架防护: 现代框架自动转义、Vue/React 防护

CSRF (跨站请求伪造) ​

  • 原理: 在其他网站执行未授权操作、借用用户身份
  • 条件: 用户登录、浏览器自动认证、操作敏感
  • 危害: 转账、修改密码、删除数据

防护方法 ​

  • CSRF Token: 表单隐藏字段、验证 Token 有效性
  • SameSite Cookie: 限制跨域 Cookie 发送
  • 验证来源: Origin/Referer 检查、二次认证

SQL 注入 ​

攻击方式 ​

  • 拼接 SQL: 未经处理的用户输入、直接拼接
  • 注入恶意 SQL: 修改查询逻辑、删除数据、提升权限

防护方案 ​

  • 参数化查询: PreparedStatement、绑定变量、SQL 引擎处理
  • ORM 框架: Hibernate、SQLAlchemy、自动防护
  • 输入验证: 类型检查、长度限制、白名单
  • 最小权限: 数据库用户权限限制、受限操作

身份验证与授权 ​

身份验证 ​

  • 密码: 强密码、盐值哈希（bcrypt、PBKDF2）、多因素认证
  • Token: JWT、OAuth、长期凭证、无状态验证
  • 生物识别: 指纹、面部识别、物理安全

授权管理 ​

  • RBAC: 角色生成权限、层次关系、成本权衡
  • ABAC: 属性权限控制、灵活精细、规则复杂
  • 权限验证: 每个请求验证、黑名单、白名单

加密与哈希 ​

对称加密 ​

  • AES: 高性能、广泛应用、密钥管理困难
  • 应用: 数据加密、传输加密、备份加密

非对称加密 ​

  • RSA: 密钥配对、数字签名、性能低
  • ECC: 更小密钥、相同强度、计算高效

哈希函数 ​

  • MD5/SHA1: 已破解、不建议使用
  • SHA-256/512: 安全、广泛使用
  • PBKDF2/bcrypt: 密码哈希、故意慢、盐值处理

安全通信 ​

HTTPS/TLS ​

  • SSL/TLS: 加密传输、身份认证、完整性检查
  • 证书: CA 签证、信任链、有效期限制
  • 协议版本: TLS 1.2+、向下兼容风险、性能平衡

密钥管理 ​

  • 生成: 强随机数、足够长度
  • 存储: 密钥库、HSM 硬件、权限隔离
  • 轮换: 定期更换、泄露后立即更换
  • 备份: 安全保存、灾难恢复

OWASP 安全清单 ​

常见漏洞顶 10 ​

  1. 注入: SQL 注入、命令注入、LDAP 注入
  2. 损坏身份验证: 密码管理、会话管理失效
  3. 敏感数据泄露: 未加密、传输安全、日志记录
  4. XML 外部实体 (XXE): XML 解析、外部实体引用
  5. 访问控制缺失: 权限检查不足、直接对象引用
  6. 安全配置错误: 默认密码、不必要服务、调试模式
  7. XSS: 跨站脚本、反射存储 DOM
  8. 不安全反序列化: 恶意对象、代码执行
  9. 已知漏洞组件: 过期库、NPM 漏洞
  10. 监控和日志缺失: 审计不足、告警缺人、溯源困难

安全实践 ​

代码审查 ​

  • 静态分析: 工具扫描、模式匹配、缺陷检测
  • 代码审查: 同行评审、知识共享、质量控制
  • 依赖扫描: NPM audit、OWASP DependencyCheck

安全测试 ​

  • 渗透测试: 模拟破解、寻找漏洞、改善防御
  • 模糊测试: 随机输入、异常处理、边界条件
  • 安全扫描: OWASP ZAP、Burp Suite、自动化测试

安全部署 ​

  • 最小权限: 最少权限原则、权限隔离
  • 隔离: 网络划分、容器隔离、虚拟化
  • 监控告警: 异常检测、入侵告警、应应机制

安全工具 ​

  • 扫描: Nmap、Burp Suite、OWASP ZAP
  • 分析: Checkmarx、SonarQube、Fortify
  • 依赖: npm audit、OWASP DependencyCheck、Snyk
  • 加密: OpenSSL、Cryptography、PyNaCl

最佳实践 ​

安全文化 ​

  • 意识培训: 员工安全培训、定期更新
  • 标准规范: 安全编码、审核流程、政策制定
  • 应急预案: 事件响应、灾难恢复、沟通计划

持续改进 ​

  • 漏洞追踪: 缺陷管理、优先级评估、及时修复
  • 补丁管理: 定期更新、兼容性测试、验证有效性
  • 审计日志: 变动记录、异常检测、合规证明
