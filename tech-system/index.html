技术体系 ​

本站“技术体系”仅面向 初级 / 中级开发，用来梳理学习路线与知识地图。偏 中高级/高级 的内容（如分布式一致性、云原生、LLM 微调、数仓/大数据、系统架构模式专题等）会暂时下线。

🏗️ 核心技术栈 ​

后端技术栈 ​

  • Java/Spring 生态 - Spring Boot、常用组件、工程化
  • Python 生态 - Django、Flask、FastAPI
  • Node.js 生态 - Express、NestJS
  • Go 语言 - 基础语法、并发、Web 开发
  • 物联网项目学习路线 - 设备接入、数据上报、在线状态、可视化

工业自动化（PLC） ​

  • 常见 PLC 品牌与选型 - 西门子、三菱、欧姆龙等主流品牌对比与选型指南
  • 西门子 S7 详解与 Python 采集 - S7 协议、内存结构、python-snap7 实战
  • S7 速查手册 - 常用 API、地址对照、快速示例

前端技术栈 ​

  • Vue.js 生态 - Vue 3、组合式API、Vuex/Pinia
  • React 生态 - React 18、Hooks、Redux、Next.js
  • 前端工程化 - Webpack、Vite、TypeScript、测试
  • 移动端开发 - React Native、Flutter、PWA
  • 跨平台方案 - Electron、Tauri、WebAssembly

数据库与存储 ​

  • 关系型数据库 - MySQL、PostgreSQL优化
  • NoSQL 数据库 - MongoDB、Redis、Elasticsearch
  • 缓存系统 - Redis集群、缓存策略、性能优化

数仓/大数据、分布式存储等偏中高级内容，暂时下线。

🔌 系统集成与设备通信（HTTP / 工控） ​

面向 多语言服务对接 与 PLC 数采 等场景：广义都属于「数据通信」，但 HTTP 偏 应用层 API，Modbus TCP 偏 工控设备层协议。

  • 跨语言调用：方式选型与常见实现 — HTTP/gRPC/MQ/FFI/子进程等是否可行、怎么选
  • HTTP 跨语言协作：Node.js ↔ Python — REST/JSON、网关与算法服务拆分、常见 Express/axios 踩坑
  • Modbus TCP：Node.js ↔ PLC — 从站、寄存器/线圈、502 端口、联调清单与安全注意

可与 物联网项目学习路线 一起看：设备接入、上报与可视化常与上述通信方式组合使用。
