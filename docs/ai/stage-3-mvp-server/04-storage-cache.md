# 04 存储与缓存：文件/SQLite/MySQL/Redis 怎么选

> 目标：根据你现在“学习 + 小工具”阶段，选一个成本最低、最好维护的方案。

## 1）第一版推荐：本地文件 or SQLite

- **本地文件（JSON）**：
  - 优点：实现快、零依赖
  - 缺点：并发与更新控制弱
- **SQLite**：
  - 优点：单文件数据库、查询方便
  - 缺点：多人协作/高并发不适合

## 2）什么时候上 MySQL / Redis？

- 你要做成多人可用服务、或数据量明显上来时：
  - MySQL：结构化持久化
  - Redis：热点缓存、限流、任务状态

## 3）验收标准

- 你的 `/context` 响应稳定且延迟可接受
- 你能解释为什么现在先不用复杂中间件
