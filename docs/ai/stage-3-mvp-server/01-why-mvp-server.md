# 01 为什么需要中台：从“模型乱搜”到“数据可控”

> 目标：理解这一阶段的意义：**让模型基于你“整理过的上下文”输出，而不是随机抓取/胡乱联想**。

## 1）没有中台的典型问题

- 上下文不稳定：同一问题多次回答差异大
- 数据不可追溯：你不知道它依据了哪段材料
- 成本不可控：频繁调用外部 API + LLM

## 2）MVP Server 做什么（一句话）

把复杂数据源变成统一接口：

- 你控制“数据从哪来、长什么样、怎么更新”
- Agent 只负责“拿到干净数据 → 调模型生成结果”

## 3）你最适合的中台场景

围绕你这套 VitePress 知识库：

- 接入：扫描 `docs/`，抽取标题/小节/标签/摘要
- 输出：`/context?module=database` 返回相关内容片段（结构化）
