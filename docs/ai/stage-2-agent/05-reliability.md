# 05 多步任务与可靠性：重试、超时、日志、幂等

> 目标：让你的 Agent 从“能跑”变成“更稳定、可复用”，避免一报错就中断或写坏文件。

## 1）你一定会遇到的问题

- 模型超时/限流
- 输出格式漂移（不按模板）
- 多次运行生成重复文件
- 写入中途失败导致半成品

## 2）最低成本的工程化加固

- **超时 + 重试**：对 LLM 调用做超时，失败重试 1～3 次
- **输出校验**：检查是否包含必须栏目（题目/答案/追问/易错点）
- **幂等**：同一输入生成同一路径时，不覆盖，改成追加时间戳
- **日志**：记录一次任务的输入、输出路径、字数、耗时、错误

## 3）今日练习（40 分钟）

- 给你的 Agent 加上输出校验：不合格就自动让模型“按模板重写”
