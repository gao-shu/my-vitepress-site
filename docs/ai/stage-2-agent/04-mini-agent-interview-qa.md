# 04 最小闭环：面试题生成 Agent（读 → 生成 → 写）

> 目标：做一个真正“有用”的 Agent：输入模块名，它去读你的笔记，然后生成题库并写回一个新页面。

## 1）功能定义（MVP）

- 输入：模块名（例如 `数据库`）
- 行为：
  1. 读取该模块 1～3 篇核心笔记
  2. 生成 10 道面试题（含答案/追问/易错点）
  3. 写入 `docs/generated/<模块>-interview-qa.md`
- 输出：生成文件路径

## 2）Prompt 要求（关键）

- 只允许基于输入材料，不要编造不在材料中的细节
- 输出必须是 Markdown 固定模板（题目/答案/追问/易错点）

## 3）验收标准

- 你对 `docs/database/overview.md` 跑一遍，生成一篇题库页面
- 页面在 VitePress 能正常打开
