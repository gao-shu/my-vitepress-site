# 01 Agent 是什么：LLM + Tools + 循环

> 目标：把 Agent 想清楚，避免“上来就学框架，越学越乱”。

## 1）一句话理解 Agent

**Agent = LLM（理解/规划） + Tools（执行动作） + Loop（反复执行直到完成）**。

LLM 负责“想”和“说”，Tools 负责“做”：

- 查数据（HTTP/DB）
- 读写文件（Markdown/JSON）
- 调用你自己的业务接口

## 2）Agent 的最小闭环（你必须做出来）

1. 用户输入任务（例如：给我生成“Redis 高频题”）
2. LLM 决定要用什么工具（例如：读取某篇笔记）
3. 工具执行并返回结果（读到文本）
4. LLM 基于结果生成输出（题目/答案）
5. 再调用工具写回文件（生成新 Markdown）

## 3）你现在最适合做的 Agent 场景

围绕你这个 VitePress 知识库：

- 输入：模块名（Java/数据库/Redis/框架/物联网）
- 输出：一篇新页面（题库/学习计划/口语化回答模板）

> 这类 Agent 最容易落地，也最适合你“求职面试”主线。
