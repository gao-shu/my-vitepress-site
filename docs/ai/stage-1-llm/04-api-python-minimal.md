# 04 API 入门：用 Python 写一个最小调用脚本

> 目标：把“会聊天”升级成“会写脚本调用”，为后面 Agent/工具链打基础。

## 1）你要实现的最小功能

- 输入：一段文本（命令行/文件）
- 输出：模型生成的 Markdown（保存到 `outputs/`）

## 2）推荐项目结构（最小）

- `main.py`
- `inputs/`
- `outputs/`

## 3）依赖与运行（建议用 uv）

- 用 `uv add ...` 安装你需要的 SDK（按你选择的平台来）
- 用 `uv run main.py` 运行

> 这一篇不强绑定某一家模型 SDK，你按你实际选择的平台 Quickstart 抄一版即可。

## 4）关键点（写脚本时别踩坑）

- 把 Key 放环境变量，不要硬编码
- 控制输出格式：让模型只输出 Markdown
- 做好失败重试/超时（后面再补也行，但要有意识）

## 5）今日练习（30～60 分钟）

- 让脚本支持两种输入：
  - 直接从命令行传入文本
  - 从 `inputs/xx.md` 读取
- 输出固定写到 `outputs/summary.md`
