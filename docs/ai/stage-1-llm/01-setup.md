# 01 环境与账号准备（Web + API）

> 目标：在 30～60 分钟内完成“能用 Web 端 + 能用 API 调用”，为后续脚本/小工具做准备。

## 1）你需要准备什么

- **一个稳定可用的大模型 Web 端**：用于日常学习与 Prompt 迭代。
- **一个可用的 API Key**：用于写脚本/做小工具。
- **本机 Python 环境**：推荐用 `uv`（更快、更省心）。

## 2）Python + uv（推荐）

### 2.1 安装与校验（思路）

- 安装 `uv` 后，保证能在终端执行：
  - `uv --version`
  - `python --version`

### 2.2 建一个最小项目目录

在任意目录创建项目（示例）：

- `uv init`
- `uv venv`

> 你后面所有小脚本都建议放到一个独立目录里，别污染你的 VitePress 项目。

## 3）API Key 与环境变量（必须）

推荐用环境变量存 Key（不要写进代码、不要提交 Git）：

- Windows（示例思路）：在系统环境变量里新增 `OPENAI_API_KEY`（或你使用的平台对应变量名）。
- 也可以用 `.env` 文件（结合 `python-dotenv`），但注意 `.env` 要加入 `.gitignore`。

## 4）最小可验证动作（今天就做完）

- 用 Web 端做 3 个任务：
  - 让模型把一段技术文章总结成 5 点
  - 让模型把一段“面试回答”改写得更口语
  - 让模型按固定 Markdown 模板输出
- 为阶段一后续文章预留一个目录：
  - `outputs/`：存放生成的 Markdown
