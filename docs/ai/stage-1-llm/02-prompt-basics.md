# 02 Prompt 基础：角色/目标/输入输出约束

> 目标：把“会问问题”升级成“可复用 Prompt 模板”，让输出稳定、可控。

## 1）Prompt 的最小结构（背下来）

- **角色（Role）**：你希望模型扮演谁
- **目标（Goal）**：你要它完成什么任务
- **输入（Input）**：你提供什么材料
- **输出（Output）**：必须用什么格式输出（强约束）
- **边界（Constraints）**：不能做什么、不要编造、不要废话

## 2）三套你立刻就能用的模板

### 2.1 面试题生成（基于一段笔记）

把你笔记贴进去，用这个模板：

- 角色：资深 Java 面试官
- 目标：生成 10 道面试题 + 标准答案
- 输出：Markdown，按“问题/回答/追问/易错点”

### 2.2 学习总结（基于一篇文章）

- 角色：技术文档编辑
- 目标：提炼 5 点要点 + 2 个例子 + 1 个常见坑
- 输出：Markdown，三级标题 + 列表

### 2.3 简历优化（基于一段经历）

- 角色：技术面试官 + 简历教练
- 目标：改写为“动作 + 技术栈 + 结果”
- 输出：3 条可放简历的 bullet

## 3）输出格式强约束（最重要）

你只要记住一句：**让模型按你要的格式输出，不要让它自由发挥。**

常用约束：

- “必须输出 Markdown”
- “必须包含：标题/步骤/结论”
- “每条不超过 2 行”
- “不要出现‘可能/大概’”

## 4）今日练习（20 分钟）

- 选你 `docs/` 里任意一篇文章
- 用“学习总结模板”让模型输出
- 你手动微调 Prompt 3 次，观察输出变化
