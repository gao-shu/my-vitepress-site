# 03 Prompt 进阶：Few-shot、结构化输出、评估与迭代

> 目标：让输出更稳定、更一致，减少“同一句话问三遍三种答案”的波动。

## 1）Few-shot：给 1～2 个示例，效果立刻提升

使用方式：

- 在 Prompt 中先给 **输入示例** 与 **期望输出示例**
- 再贴入你的真实输入

适合场景：

- 固定模板输出（简历 bullet、面试题模板、日报模板）
- 需要统一口径（同一风格、同一栏目）

## 2）结构化输出：让模型输出“可解析”的结果

优先两种：

- **Markdown 固定模板**（你最常用）
- **JSON**（方便程序处理，后面 Agent/工具调用会用到）

示例目标（描述即可）：

- “只输出 JSON，不要输出任何解释文字”
- “字段必须包含：title, points, pitfalls”

## 3）评估与迭代：别凭感觉，做个小对比

最小评估办法：

- 用同一段输入，准备 2～3 个 Prompt 版本
- 让模型输出后，你用 3 个标准打分：
  - **完整性**：是否漏点
  - **可读性**：是否能直接用
  - **一致性**：多次运行是否差不多

## 4）今日练习（30 分钟）

- 选一个你常用场景：比如“生成面试题”
- 用 Few-shot 把输出模板固定下来
- 再把输出从“纯文字”改成“Markdown 固定栏目”
