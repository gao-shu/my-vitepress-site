# 阶段二：MVP Server 与数据中台规划【让模型“吃对数据”】

> 这一篇规划的是：**怎么搭一个最小可用的“数据中间层”，专门给大模型/Agent 提供干净、统一的上下文数据**。

---

## 1. 学习目的与适合人群

- 你已经会基础的 LLM 调用，希望：
  - 不再“每次都让模型自己上网搜”，而是先经过你整理的数据。
  - 为后面做“投资助手 / 学习助手 / 公司内部问答系统”打基础。

这一阶段**非常适合你现在的状态**：  
可以把你自己的面试笔记、学习资料，看作一个“小型数据中台项目”来练手。

---

## 2. 这一阶段的核心目标

1. 会设计简单、统一的 REST API（比如 `/context`、`/articles`）。  
2. 会从多种数据源（本地 Markdown、数据库、外部 API）拉数据并**清洗成统一结构**。  
3. 能写一个小服务，让 LLM/Agent 调用这个服务拿到“整理好的上下文”。

---

## 3. 时间规划建议（约 2～3 周）

### 第 1 周：API 设计与单一数据源整合

- 学习/复盘内容：
  - REST API 设计基础（路径命名、状态码、GET/POST 语义）。
  - 选一个后端技术栈（推荐你熟悉的 Spring Boot）。
- 实战任务：
  - 写一个最简单的 `/health` 接口，熟悉项目结构。
  - 再写一个 `/articles` 接口，返回几条**写死的模拟数据**，格式类似：
    ```json
    [
      {
        "id": 1,
        "title": "数据库事务与锁",
        "tags": ["database", "transaction"],
        "content": "这里是正文或摘要...",
        "source": "local-notes"
      }
    ]
    ```

### 第 2 周：接入真实数据源（本地笔记 / Markdown / 数据库）

- 数据源选择（可从简单到复杂）：
  - 第一层：从你的 VitePress 项目里的 **Markdown 文件** 读取内容。  
  - 第二层：把部分内容导入到数据库（如 MySQL/SQLite），便于查询。
- 实战任务：
  - 编写一个简单脚本或服务逻辑：
    1. 扫描某个目录（如 `docs/database`）。  
    2. 把文件名、标题、摘要抽出来。  
    3. 存到内存/数据库，用统一结构暴露出 `/articles` 或 `/context` 接口。

### 第 3 周：让 LLM 通过这个 MVP Server 获取数据

- 目标：
  - 让你的 LLM 调用不再直接“死问模型”，而是：
    1. 先请求你的 `/context` 接口拿到相关资料。  
    2. 再把这些资料作为 `context` 一并发给模型，让模型基于这些内容回答。
- 实战任务：
  - 写一个简单流程（可以单函数实现）：
    ```text
    用户问题 → 调用 /context?topic=数据库事务 → 得到相关文章列表 →
    从中拼出一段上下文 → 调用 LLM → 生成回答
    ```

> 这一套下来，你就有了一个**“基于自己知识库的问答系统雏形”**，非常适合后面升级成 RAG/知识库项目。

---

## 4. 针对不同学习目的的侧重点

### 目的 A：只希望提升自己项目里的“数据整理能力”

- 可以只做到：
  - 会从 Markdown/数据库拉数据。
  - 会设计 1～2 个统一接口返回整理好的 JSON。
- 不强求你把 LLM 整合进来，但建议留一个“对接口”的位置，后续随时可以接上。

### 目的 B：想做一个“学习/面试助手”产品雏形

- 推荐目标：
  - 完成一个**“我的面试知识中台”**：
    - 后端：Spring Boot 服务 + `/context` 接口。  
    - 数据：来自你当前的 VitePress docs。  
    - 前端：一个简单页面，输入模块名或关键词，展示相关资料 + 一键发给 LLM 总结。

### 目的 C：以后想做 AI 后端/平台型岗位

- 在 B 的基础上，多注意：
  - API 设计规范、错误码规范。
  - 日志埋点：记录什么问题查了什么 context，后续方便调优。
  - 简单的鉴权（即使只是一个固定 token 也行）。

---

## 5. 如何与现有 Java/数据库学习主线结合

- 你当前网站的所有内容，其实就可以视为一个“小型知识库”，  
  阶段二的 MVP Server 就是把这些知识库**“结构化 + 服务化”**的过程。

建议这样结合：

1. 先选一个模块（比如数据库）：  
   - 把 `/database` 下的文章都纳入 `/context` 接口。  
2. 再选一个模块（比如 Java 并发）：  
   - 同样做一层整理。  
3. 最后做一个“综合问答”的小页面：
   - 你输入问题，它先在本地知识库里找相关内容，再让 LLM 帮你回答。

这时候，你的 AI 学习就不再是“额外多学一门”，而是**直接反哺你现有的面试学习体系**。
