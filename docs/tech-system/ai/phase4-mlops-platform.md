# 阶段四：部署 / 平台化 / MLOps 规划【后期选修】

> 这一篇是**完全可选的后期阶段**：当你已经有几个 AI 小项目在本地跑得不错，想进一步部署上线、做成“真正可用的服务”时，可以按这篇来规划。

---

## 1. 阶段定位与适合人群

- 适合：
  - 已经完成前面几个阶段（至少有 LLM + MVP Server 小项目）。  
  - 想把自己的“投资助手 / 学习助手 / 问答系统”部署到公网或内网给别人用。  
  - 对 DevOps / 平台化 感兴趣，希望增加简历亮点。

> 对你当前“远程 6000 初/中级岗”的目标，这一阶段是**加分项**，不是必修课。

---

## 2. 阶段目标拆解

1. 能把一个简单的 AI Web 服务打包成 Docker 镜像，并在服务器上跑起来。  
2. 知道 CI/CD 的基本流程，能用 GitHub Actions 或类似工具做个最简自动部署。  
3. 能对自己部署的服务做**基础监控**：访问量、错误率、响应时间等。

---

## 3. 时间规划建议（可拉长，按 2～4 周慢慢做）

### 第 1 周：Docker 化你的 AI 服务

- 学习/复盘：
  - Docker 基础概念：镜像 / 容器 / Dockerfile。  
  - 了解如何暴露端口、挂载配置。
- 实战任务：
  - 选你一个现有的小项目（例如“投资助手”或“面试问答助手”）。  
  - 为它写一个最简单的 `Dockerfile`，包括：
    - 拷贝代码。  
    - 安装依赖。  
    - 暴露端口 / 启动命令。
  - 在本机用 `docker run` 跑起来，确认能访问。

### 第 2 周：部署到云服务器 / 平台

- 选择一种方式：
  - 云服务器（阿里云 / 腾讯云 / 华为云轻量应用）。  
  - 或者托管平台（如 Railway / Render / Fly.io 等）。
- 实战任务：
  - 把打好的镜像推到镜像仓库（Docker Hub 或平台自带仓库）。  
  - 在服务器上拉取镜像并运行，保证：
    - 有简单的反向代理（如 Nginx）转发到你的服务。  
    - 能通过浏览器访问简单页面。

### 第 3～4 周：简单 CI/CD 与监控

- CI/CD 基础：
  - 理解流程：`git push` → 自动构建镜像 → 自动部署到测试或生产。  
  - 可以用 GitHub Actions 写一条最简 pipeline：
    - 代码 push 到 `main` 分支时，自动构建 Docker 镜像并推送到仓库。
- 监控基础：
  - 在日志中记录：请求路径、耗时、错误信息。  
  - 可选：接入一个简单的监控面板或日志服务（如 Grafana + Loki / 第三方日志平台）。

---

## 4. 针对不同学习目的的取舍

### 目的 A：只想“上线给自己或者少数朋友用”

- 做到：
  - 本机 Docker 化 + 一台轻量云服务器部署。  
  - 简单 Nginx 反代 + HTTPS（可用免费证书）。  
  - 备份配置和环境变量（避免重装后忘记怎么起）。

### 目的 B：想在简历上写“有一定 DevOps / MLOps 实践”

- 建议多做：
  - 写一份 CI/CD 脚本（哪怕是最简单的自动构建 + 手动部署也行）。  
  - 在文档中整理：
    - 架构图（浏览器 → Nginx → AI 服务 → LLM API / 向量库）。  
    - 运维流程（如何更新版本、如何回滚）。

### 目的 C：长期想往“平台工程 / AI 平台”方向发展

- 这一阶段只是一个起点，后续可以扩展到：
  - Kubernetes / Helm / 服务网格等。  
  - 模型版本管理、特征存储、在线/离线混合服务等。

---

## 5. 如何与整个 AI 学习路线衔接

- 阶段一：你学会了调用大模型、写 Prompt、做小工具。  
- 阶段二：你把数据整理成 API，搭了 MVP Server。  
- 阶段三：你做了更聪明的检索（RAG）和可能的图结构。  
- 阶段四：你把这些东西**打包成“真正可访问的在线服务”**。

对你现在的主线来说，可以先把这一篇当作“未来一年内的升级选项”；  
等你把 Java / 数据库 / 框架 / 面试这些主线跑顺了，再回头按这篇规划挑一部分做，就能自然形成一个可讲的“AI + 平台实战项目经历”。
