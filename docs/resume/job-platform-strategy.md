# 投递平台与投递策略（Java 远程/异地，小公司为主）

> 目标：**提升被约面概率**（简历被看见 → 愿意聊 → 进入技术面），并减少无效投递。

---

## 1. 平台选择总览（按对你“约面效率”的优先级排序）

> 你的目标画像是：**远程/异地、小公司为主、技术面为主（自我介绍 + 项目追问）**。  
> 所以排序标准不是“平台有多大”，而是：**能不能快速触达决策人 + 反馈快不快 + 远程岗位集中不集中**。

| 优先级 | 平台 | 适合方向 | 核心特点 | 主要沟通方式 | 你该怎么用 |
| --- | --- | --- | --- | --- | --- |
| ① | BOSS 直聘 | 快速沟通、技术岗、高效反馈 | 反馈最快，很多时候能直接和负责人/老板聊 | **即时聊天**（主动打招呼） | 每天固定两次集中聊，先问 3 个确认问题，不合适就止损 |
| ② | 拉勾网 | 互联网、技术岗 | JD 质量相对高，技术栈更清晰，匹配度更容易做 | **投递 + 附言**（匹配点） | 只投匹配度高的（≥70%），附言写 3 个关键词对齐 |
| ③ | 猎聘（Liepin） | 中高级/偏资深、猎头推荐 | 你 8 年经验能吃到“猎头推荐”红利，但要像中级/资深表达 | **猎头沟通**（电话/微信/站内） | 简历用“负责/主导/落地”，沟通强调“解决问题 + 交付” |
| ④ | 前程无忧（51job） | 全行业岗位、传统公司/外包/小公司 | 岗位量大但噪声多，适合补量 | **投递 + 站内信** | 投递同时发一句确认远程/异地的站内信，提高回复率 |
| ⑤ | 智联招聘 | 各级岗位、综合类 | 岗位多但重复多，需要强筛选 | **投递 + 筛选** | 同公司重复岗位只投一个，关键词排雷（驻场/出差等） |
| ⑥ | 鱼泡直聘（鱼泡网/鱼泡人才） | 小微企业、本地/异地、外包/项目制 | 信息密度高但质量参差，适合“快验证” | **即时沟通/电话**（更偏快） | 先问清结算/周期/合同/远程，避免投入太多时间 |

### 1.1 一句话总结每个平台的“打法”

- **BOSS**：把投递变成“短对话成交”（你最该重视）。  
- **拉勾**：用“匹配度投递 + 附言对齐”吃到 JD 质量的红利。  
- **猎聘**：用“资深表达 + 猎头渠道”拿到更高质量的技术面机会。  
- **51job/智联**：补量用，但必须筛（否则浪费时间）。  
- **鱼泡**：快筛快聊快验证，不要长聊（防噪声/防踩坑）。

## 2. 你的投递核心策略（适配：小公司、技术面为主、远程/异地）

### 2.1 一句话定位（你要在简历/开聊里反复出现）

- **定位**：`Java 后端 / Java 全栈（可远程）｜8 年经验｜Spring Boot/Cloud + MySQL/Redis + Docker｜能独立交付`

这句话的作用是：让对方快速判断“你能不能直接上手干活”，减少无效沟通。

### 2.2 目标岗位拆分（避免“什么都投”）

建议你把投递拆成两条线（两个简历版本都可以共用主体，只改标题与技能强调点）：

- **路线 A：Java 后端（主线）**  
  - 关键词：`Spring Boot`、`MySQL 优化`、`Redis 缓存`、`微服务组件（Nacos/Feign）`、`Docker`  
- **路线 B：Java 全栈（备线）**  
  - 关键词：`Java + Vue/TS`、`前后端分离`、`独立交付`、`部署上线`  

> 不建议同时对外强调 “Java 全栈 + React + Vue + AngularJS + Netty + ES + MQ 全都会”，会显得发散。要“像一个能落地的人”，而不是“技能堆叠的人”。

## 3. 简历被选中概率提升：4 个最有效动作

### 3.1 标题/期望要“对齐 JD”

你简历顶部建议这样写（更利于筛选系统/HR秒懂）：

- **求职意向**：`Java 后端（可兼前端/全栈）`
- **工作方式**：`远程 / 异地（可接受）`
- **期望薪资**：`6k-10k（可谈，以远程与成长空间优先）`

### 3.2 工作经历里补“技术亮点钩子”（每段 1～2 个就够）

你现在的经历描述偏“做了什么”，建议补成“做了什么 + 解决什么问题 + 用了什么技术”。  
不需要写很多，但每段最好有 1 个可追问点，例如：

- **MySQL**：慢查询排查（慢日志/EXPLAIN）+ 索引优化
- **Redis**：数据类型/持久化/分布式锁（能讲清原理与注意事项）
- **Docker**：容器化部署、Compose 编排

（这些钩子对应的面试准备，你已经有：[专业技能面试准备](./skills-interview-prep.md)）

### 3.3 投递前 30 秒“岗位匹配标注”

你在投递/开聊时，把 JD 里的 3 个关键词对齐到你简历的 3 个点，效果比海投强很多：

- 例：JD 写 `Spring Boot + Redis + Nacos`  
  - 你开聊就写：`我近4年主要用 Spring Boot/Cloud Alibaba（Nacos/Feign）+ Redis + MySQL，能独立交付并负责过 Docker 部署。`

### 3.4 只投“高通过率岗位”（小公司尤其重要）

**直接跳过**这些信号（省时间就是提高命中率）：

- JD 写得很虚（“积极向上、吃苦耐劳”占比大），技术栈不清晰
- 职责和要求明显不匹配（要你做 Java，却写“精通前端动效/小程序”）
- 薪资写“面议”但要求“全能 + 加班 + 驻场”

## 4. 各平台具体打法（怎么投、怎么聊、怎么筛）

### 4.1 BOSS 直聘（最适合你：快聊快验证）

**目标**：把“简历投递”变成“短对话成交”。  

**动作**：

- **筛选**：优先筛 `1-20人/20-99人`、`Java 后端/全栈`、`远程/异地`、`最近活跃`  
- **节奏**：每天固定 2 个时间段集中沟通（例如 10:30、20:30），避免碎片化消耗
- **策略**：先问 3 个关键确认问题（不合适就止损）

**开聊模板（可直接复制）**：

> 你好，我是张孟杰，8 年 Java 后端/全栈经验，最近 4 年主要是 Spring Boot/Cloud Alibaba（Nacos/Feign/Xxl-job）+ MySQL/Redis，也做过 Docker/Linux 部署。  
> 我期望远程/异地（可灵活），想确认下：  
> 1）这个岗位是否支持远程/异地？  
> 2）主要技术栈是 Spring Boot 单体还是微服务？是否用 Redis/MySQL？  
> 3）面试流程是否以技术面为主（自我介绍 + 项目提问）？

**你要引导对方问你的点（钩子）**：

- `我做过慢查询排查和索引优化（慢日志 + EXPLAIN）`
- `我熟悉 Redis 的数据结构/持久化/分布式锁实现与注意事项`
- `我能独立 Docker 部署上线`

---

### 4.2 51job（岗位多，关键是“筛+少而准”）

**策略**：

- 优先投：`Java 后端`、`Spring Boot`、`MySQL/Redis` 明确的岗位
- “远程/异地”不明确的，**投递同时发站内信**确认工作方式
- 重点投递：近 7 天新发布、回复率高的岗位

**站内信模板**（更短，更像确认信息）：

> 你好，我是 8 年 Java 后端/全栈，技术栈 Spring Boot/Cloud + MySQL/Redis + Docker。请问该岗位是否支持远程/异地？技术面主要围绕项目经验展开吗？

---

### 4.3 智联招聘（岗位重复多，用“关键词排雷”）

**策略**：

- 用关键词排雷：`驻场`、`外包`、`出差频繁`、`996`（按你意愿）
- 同一家公司的重复岗位，**只投一个最匹配的**
- 看到“岗位描述与要求严重不一致”的直接跳过

---

### 4.4 拉勾网（JD 质量高，适合“匹配度投递”）

**策略**：

- 只投匹配度 $\\ge 70\\%$ 的岗位（技术栈对齐、经验要求对齐）
- 投递时附一句“匹配点”最有效（3 个关键词对齐）

**投递附言模板**：

> JD 提到 Spring Boot/Redis/Nacos，我近 4 年主要用 Cloud Alibaba（Nacos/Feign/Xxl-job）+ Redis + MySQL，能独立交付并负责过 Docker 部署，想进一步聊下项目匹配。

---

### 4.5 鱼泡直聘（快筛快聊：防“信息噪声”）

**策略**：

- 先确认 3 件事：`是否远程/异地`、`薪资范围`、`是否技术面为主`
- 对“外包/项目制”岗位：问清楚结算方式、周期、是否签合同（避免踩坑）
- 不要长聊，**快速验证**：不合适就止损

---

### 4.6 猎聘（适合你：8 年经验要“像中级/资深”）

**策略**：

- 简历关键词要更偏“负责/主导/落地”，不要全是“参与/负责开发”
- 接到猎头沟通时：用“你能解决什么问题”来介绍自己（性能、交付、协作）

**猎头沟通 30 秒版自我介绍**：

> 我 8 年 Java 后端/全栈，最近 4 年以 Spring Boot/Cloud Alibaba 为主，做过 CRM/ERP 这类核心业务系统，也独立交付过物联网项目。我的优势是能把项目从需求到上线跑通，并且在 MySQL/Redis 性能与稳定性上能落地优化，另外熟悉 Docker/Linux 部署。

## 5. 提升“被约面”的关键：投递节奏与复盘

### 5.0 远程/居家岗位“搜索与筛选”SOP（关键词 + 城市打法）

> 你的目标很明确：找 **Java 远程/居家**（含全栈远程/居家）。  
> 这类岗位在各平台通常不是一个独立标签，更多是“写在 JD 里/写在沟通里”，所以关键在于：**用关键词把它捞出来 + 用城市热点提高命中率 + 用固定节奏反复操作**。

#### 5.0.1 关键词库（你给的 4 个 + 推荐补充）

你给的关键词（保留）：
- `java远程`
- `java居家`
- `全栈远程`
- `全栈居家`

建议补充（更常见的表达）：
- `远程` / `居家` / `在家办公`
- `异地` / `异地办公`
- `远程办公` / `居家办公`
- `远程开发` / `远程工程师`
- `远程兼职`（只在你接受兼职/项目制时用）

> 注意：平台搜索大多对空格/大小写不敏感，但对“同义词”不等价；**同一个平台要多组关键词轮换**，别只搜一种。

#### 5.0.2 组合公式（直接照抄去搜）

你可以按“岗位方向”分两条线搜（和第 2.2 节一致）：

- **路线 A：Java 后端（主线）**
  - 组合 1：`Java 远程 Spring Boot`
  - 组合 2：`Java 居家 Spring Boot`
  - 组合 3：`Java 远程 MySQL Redis`
  - 组合 4：`后端 远程 Java`
- **路线 B：Java 全栈（备线）**
  - 组合 1：`Java 全栈 远程 Vue`
  - 组合 2：`全栈 远程 Java Vue`
  - 组合 3：`全栈 居家 Java`

如果平台支持“排除词/过滤”，优先排除这些（按你的偏好取舍）：
- `驻场`、`外包驻场`、`坐班`、`必须到岗`、`出差`
- `Java培训`、`实习`（避免噪声）

#### 5.0.3 热门城市前十：为什么要“城市化搜索”

即使你找远程，也建议你把搜索城市重点放在岗位最集中的城市，因为：
- 远程岗位往往“挂在总部城市”，HR/老板也更集中
- 同样关键词下，热门城市的“新发布 + 活跃沟通”密度更高

你要覆盖的前十热门城市（按你给的）：
**上海、北京、深圳、广州、杭州、成都、南京、武汉、苏州、重庆**

打法建议：
- **先搜一线/新一线（上海/北京/深圳/广州/杭州/成都）**：远程岗位密度高、技术栈更清晰
- **再补二线热门（南京/武汉/苏州/重庆）**：竞争相对小，容易聊起来

#### 5.0.4 平台级“怎么搜”（把关键词 + 城市落到操作）

> 共同原则：每个平台都要做两件事 —— **保存搜索条件**（或收藏岗位）+ **只看近 3 天/近 7 天**（提高回复速度）。

- **BOSS 直聘**
  - **城市策略**：按“城市”逐个切换（上面 10 城轮换），每个城市用 2~3 个关键词组合
  - **筛选条件**：`最近活跃`、`1-99人/100-499人`、`3-10年/5-10年`（贴合你年限）、`后端/全栈`
  - **关键动作**：把“搜到岗位”变成“发起对话”，不要只投递不说话（见 4.1 开聊模板）

- **拉勾**
  - **城市策略**：优先上海/北京/深圳/广州/杭州/成都；每城只投匹配度高的（≥70%）
  - **关键词打法**：用 `Java + 远程/居家` 作为主关键词，再用 `Spring Boot` 缩小范围
  - **关键动作**：投递附言固定 1 句“匹配点”，否则和海投差不多（见 4.4 模板）

- **猎聘**
  - **城市策略**：先一线/杭州/成都；猎头更集中在这些城市
  - **关键词打法**：`Java 远程` + `后端` / `全栈`；如果能选“工作地点/期望城市”，按 10 城轮换
  - **关键动作**：与猎头沟通时先确认“是否支持远程/异地”，避免被引导去驻场

- **51job / 智联**
  - **城市策略**：建议你用“城市 + 远程/居家关键词”一起搜；因为两者的“远程标签”不稳定，靠文本命中更可靠
  - **关键词打法**：优先 `Java 远程`、`Java 居家`、`远程 Spring Boot`
  - **关键动作**：投递后补一句站内信确认远程（见 4.2 模板）；不确认，回复率会明显下降

- **鱼泡**
  - **城市策略**：以你愿意合作的城市为主（热门城市先），因为鱼泡里“项目制/外包”更常见
  - **关键词打法**：`Java 远程` / `Java 全栈 远程`，并在沟通里第一句话确认“是否真远程”

### 5.1 每日投递节奏（推荐）

- **每天**：10～20 个“高匹配岗位”  
- **每周**：复盘一次（哪些平台回复率高、哪些关键词命中率高）

### 5.1.1 每天几点操作一遍？（给你一套固定作息）

> 目标：既保证“新岗位第一时间看到”，又避免全天碎片化消耗。

推荐你每天 2～3 个固定时段（按你个人作息可前后挪 30 分钟）：

- **上午 09:30（15-25 分钟）**：刷“新发布/近 24 小时”，10 城里选 2 城快速扫一遍（优先上海/北京/深圳/广州/杭州/成都轮换）
- **中午 12:30（10-15 分钟，可选）**：只做“回消息 + 约面确认”，不新增大量投递（避免午休耗散）
- **晚上 20:30（25-40 分钟）**：集中开聊 + 跟进已读未回 + 补投递（这是最容易约到技术面的一段）

如果你只想每天“操作一遍”，就选 **20:30** 这一档（晚间活跃度通常最高）。

### 5.1.2 你每天这样做一遍，会怎样？（7 天可见的变化）

你按上面节奏执行 7 天，通常会出现这些可预期的变化：

- **第 1-2 天**：你会明显感觉“远程/居家岗位并不多”，但你能快速识别哪些 JD 只是“写着远程、实则到岗/驻场”
- **第 3-4 天**：你会发现某 2-3 个城市 + 某 2-3 组关键词的命中率明显更高（这时开始收敛关键词与城市）
- **第 5-7 天**：你的对话池会形成（已读/未回/可约面），回复率和约面率开始稳定

建议你用三个阈值做调整（很实用）：
- **已读率 < 30%**：改简历标题/平台标题关键词（把“远程/异地”放进标题），并收敛岗位方向
- **回复率 < 10%**：改开聊首句（更短、更像“确认信息”），并增加“城市轮换 + 关键词轮换”
- **约面率 < 30%（在有回复的前提下）**：把对话收敛到“最近项目 + 可讲清的模块”，别扩散到不熟的点

### 5.2 复盘指标（只看这 3 个）

- **已读率**：简历是否被点开（低 → 标题/关键词不对齐）
- **回复率**：是否愿意聊（低 → 开聊话术/匹配点不清）
- **约面率**：是否进入技术面（低 → 项目亮点/钩子不够硬）
