## ⚠️ 简历可能存在的问题点

### 1. **技术栈跨度大，但深度可能不够**

**问题**：
- 从 Struts2/Hibernate（2016-2019）到 Spring Cloud Alibaba（2020-2024），技术栈跨度大
- 面试官可能会问：为什么换技术栈？你对哪个最熟悉？

**准备方向**：
- 强调技术演进：从传统SSH到微服务架构的学习路径
- 突出当前最熟悉的技术栈：Spring Boot/Cloud、MyBatis、Redis
- 准备解释：为什么从Struts2迁移到Spring Boot（性能、维护性、生态）

### 2. **项目描述比较泛，缺少具体技术亮点**

**问题**：
- 简历中写的是“负责XX系统开发”、“完成核心功能模块开发”
- 缺少：解决了什么技术难题？性能提升了多少？用了什么技术方案？

**准备方向**：
- 每个项目准备1-2个技术亮点（用STAR法则包装）
- 准备具体数据：QPS、响应时间、优化效果
- 准备技术细节：用了什么中间件？解决了什么问题？

### 3. **工作年限与期望薪资不匹配**

**问题**：
- 8年+经验，期望薪资6-10k，可能让面试官怀疑能力或稳定性

**准备方向**：
- 强调远程工作的价值：时间自由、效率更高
- 强调全栈能力：既能做后端，也能做前端，性价比高
- 如果被问到薪资，可以说：更看重远程工作的灵活性和成长空间

### 4. **项目类型多样，但缺少高并发/大数据量场景**

**问题**：
- 项目主要是业务系统（CRM/ERP/OA/车险），可能缺少高并发场景

**准备方向**：
- 从现有项目中挖掘：数据库优化、缓存使用、异步处理
- 准备学习项目：如果有高并发场景的练习项目，可以提
- 强调：虽然业务系统，但同样需要性能优化和架构设计

### 三、技术栈深度准备

#### Spring Cloud Alibaba

**必准备问题**：
1. Nacos作为注册中心和配置中心的区别？
2. Nacos的CP还是AP模式？
3. Feign和RestTemplate的区别？Feign的负载均衡？
4. Sentinel的限流原理？
5. Seata的分布式事务原理？

**如果不会，准备说**：
- "我在项目中主要用Nacos做服务注册发现，配置中心用的是Nacos的配置管理功能。对于更深入的原理，我还在学习中。"

#### Redis

**必准备问题**：
1. Redis的数据类型？各自的应用场景？
2. 缓存穿透、击穿、雪崩是什么？怎么解决？
3. Redis的持久化机制（RDB vs AOF）？
4. Redis的过期策略？
5. 分布式锁怎么实现？Redisson用过吗？

**准备案例（按真实情况选择，不要编）**：
- 在项目中用 Redis 缓存用户信息/配置类数据，减少数据库查询
- 用 Redis 做分布式锁，防止重复提交

#### MySQL

**必准备问题**：
1. 索引的数据结构？为什么用B+树？
2. 索引失效的场景？
3. 事务的隔离级别？MVCC原理？
4. 慢查询怎么排查？SQL优化经验？
5. 分库分表做过吗？

**准备案例**：
- 遇到过慢查询，通过EXPLAIN分析，发现索引失效，优化后查询时间从几秒降到毫秒
- 使用事务保证数据一致性，了解事务的传播行为

#### Docker

**必准备问题**：
1. Docker和虚拟机的区别？
2. Dockerfile的常用指令？
3. Docker的网络模式？
4. Docker Compose用过吗？
5. 容器和镜像的区别？

**准备案例**：
- 在项目中用Docker部署应用，提高了部署效率
- 写过Dockerfile，了解多阶段构建
