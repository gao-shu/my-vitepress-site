## 💡 初中级必考的高频面试题

### Java 基础 10 题

1. **什么是 Java 的三大特性？**
   - 答：封装、继承、多态
   - 深化：用实例解释多态（方法重写 vs 重载）

2. **HashMap vs Hashtable 的区别？**
   - 基础：线程安全、迭代器、扩容
   - 进阶：ConcurrentHashMap 的分段锁思想

3. **Java 多线程中的 synchronized vs volatile**
   - 基础：synchronized 保证互斥、volatile 保证可见性
   - 实战：什么场景用什么？

4. **String、StringBuffer、StringBuilder 的区别**
   - 基础：不可变性、线程安全
   - 应用：为什么 StringBuilder 更快？

5. **谈谈 Java 异常体系**
   - Exception vs Error 的区别
   - Checked vs Unchecked 异常

6. **什么是反射？有什么应用场景？**
   - 基础：动态获取类信息、调用方法
   - 应用：Spring 依赖注入、ORM 框架

7. **对象创建的过程？**
   - 类加载、内存分配、初始化、返回引用
   - 构造方法调用顺序

8. **集合框架中的 fail-fast 是什么？**
   - 原因：modCount 变化
   - 如何避免：用迭代器的 remove()

9. **ArrayList 和 LinkedList 怎么选？**
   - ArrayList：随机访问快、插删慢
   - LinkedList：插删快、访问慢

10. **什么是 Java 内存模型（JMM）？**
    - volatile 的作用、可见性、有序性

### 数据库面试题（8 道）

1. **MySQL 索引用的什么数据结构？为什么不用红黑树？**
   - B+树：多叉树、范围查询、IO 次数少
   - 聚集索引 vs 非聚集索引（InnoDB vs MyISAM）

2. **哪些情况会导致索引失效？**
   - 模糊查询 like '%xxx'、函数处理、隐式转换、OR 条件
   - 实战例子

3. **怎么分析慢查询？**
   - 使用 EXPLAIN 查看执行计划
   - 关注 type、key、rows 字段

4. **MySQL 事务的 ACID 是什么？**
   - Atomicity（原子）、Consistency（一致）、Isolation（隔离）、Durability（持久）
   - 具体怎么实现的？

5. **MySQL 四种隔离级别？**
   - READ UNCOMMITTED（脏读）
   - READ COMMITTED（不可重复读）
   - REPEATABLE READ（幻读、InnoDB 默认）
   - SERIALIZABLE（串行化）

6. **什么是 SQL 注入？怎么防护？**
   - 原因、危害
   - 解决：参数化查询、预编译、ORM 框架

7. **连接池为什么重要？**
   - 性能、资源复用、并发控制
   - HikariCP 的优势

8. **主键和唯一索引的区别？**
   - 主键非空、唯一索引可为空
   - 性能上的差异

### Spring Boot 基础题（6 道）

1. **Spring Boot 的自动配置怎么工作？**
   - @EnableAutoConfiguration、spring.factories
   - 条件装配（@ConditionalOnXxx）

2. **Spring Bean 的生命周期是什么？**
   - 创建 → 初始化 → 销毁
   - BeanPostProcessor、InitializingBean 的角色

3. **@Autowired 和 @Resource 的区别？**
   - @Autowired：按类型注入
   - @Resource：按名称注入
   - 怎么解决冲突？@Qualifier

4. **什么是 AOP？怎么实现的？**
   - 动态代理（JDK 动态代理、CGLIB）
   - 应用场景（事务、日志、权限）

5. **怎样创建一个 REST API？**
   - @RestController、@RequestMapping、@GetMapping
   - 参数验证、异常处理

6. **说说 Spring 事务？**
   - @Transactional 注解、声明式事务
   - 事务传播级别（重点：REQUIRED、REQUIRES_NEW）

### 进阶高频题

1. **什么是缓存穿透、击穿、雪崩？**
   - 穿透：查询不存在的数据 → 布隆过滤
   - 击穿：热 key 失效 → 互斥锁
   - 雪崩：大量缓存失效 → 缓存预热、热点数据重建

2. **Redis vs Memcached？**
   - 数据类型、持久化、单线程 vs 多线程
   - 应用场景

3. **分布式事务怎么保证一致性？**
   - 两阶段提交（2PC）、最终一致性
   - Seata 框架基础

4. **高并发下怎么防重复请求？**
   - 幂等性设计（自然幂等、人工幂等）
   - 令牌机制、数据库唯一索引

## 🔧 如何讲述项目经验（STAR 法则）

### 项目讲述框架

**Situation（情景）**
- 项目名称和业务背景
- 当时遇到的主要问题或挑战

**Task（任务）**
- 你的职位/职责
- 分配给你的具体任务

**Action（行动）**
- 你采取的具体行动
- 用到的技术栈
- 怎么解决遇到的问题
- 为什么这样设计？

**Result（结果）**
- 最终交付了什么？
- 带来了什么业务价值？
- 性能如何提升？

### 项目讲述示例

**情景**：一个电商平台后端项目，日均 10 万+ 请求，需要保证高可用和高性能

**任务**：负责用户模块和订单模块的开发

**行动**：
- 使用 Spring Boot + MyBatis 开发 REST API
- Redis 缓存用户信息，减少数据库查询
- 订单超时自动取消：用消息队列（RabbitMQ）+ 定时任务
- 数据库设计了合理的索引和分区

**结果**：
- 系统 QPS 从 500 提升到 5000+
- 数据库查询响应时间从 200ms 降低到 20ms
- 订单流程可靠性达到 99.9%

   - ✅ 反问面试官："我可以这样思考吗？..."
   - ❌ 瞎编或试图蒙混过关

3. **深化你的回答**
   - 面试官问"HashMap 的原理？"
   - 基础回答：结构、冲突解决

   - 说了 Redis 缓存？准备被问高可用方案

## 常见问题
